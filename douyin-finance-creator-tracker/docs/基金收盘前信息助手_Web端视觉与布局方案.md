# 基金收盘前信息助手：Web 端视觉与布局方案

**版本：** v0.1  
**定位：** 桌面 Web 优先；用于收盘前快速获取、核验与处理视频信息  
**设计方向：** Glassmorphism（深色氛围背景上的精致半透明玻璃层）

## 1. 体验目标

产品应呈现为安静、可靠、可聚焦的信息工作台，而不是刺激用户频繁交易的行情终端。

- 在 1440px 桌面宽度下，用户应能同时看到交易倒计时、信息流和待处理/风险信息。
- 14:50-15:00 的临盘新视频必须在不切换页面的情况下可见，并能一键进入原视频和字幕。
- 字幕、来源和事实依据是核心内容，视觉层次应高于装饰和动效。
- 玻璃效果用于分隔信息层级与制造空间感，不能降低文字对比度或影响长文本阅读。

## 2. 视觉语言

### 2.1 风格定义

使用深海军黑的渐变背景承载半透明玻璃面板。面板具备 `backdrop-filter: blur(20px)` 的磨砂效果、低透明填充、细腻的半透明描边与顶部内高光。薄荷绿 `#8AFFC4` 只用于关键动作、实时状态、进度和焦点，不大面积铺满页面。

背景可使用极弱的青绿/靛紫光晕和颗粒质感，制造层次；不得使用会分散长文本注意力的动态大面积流光。

### 2.2 设计令牌

| 类别 | 建议值 | 使用场景 |
| --- | --- | --- |
| 应用背景 | `#070B0F` → `#0B141A` | 页面底色，叠加低饱和径向光晕 |
| 玻璃主面板 | `rgba(16, 29, 37, 0.66)` | 主内容区、侧栏、抽屉 |
| 玻璃次面板 | `rgba(255,255,255,0.055)` | 卡片、筛选器、输入框 |
| 玻璃描边 | `rgba(218,255,240,0.16)` | 面板边缘与悬浮状态 |
| 顶部高光 | `rgba(255,255,255,0.13)` | 1px 内描边或顶部渐变 |
| 品牌薄荷绿 | `#8AFFC4` | 主按钮、实时状态、焦点环、进度 |
| 薄荷弱背景 | `rgba(138,255,196,0.12)` | 选中态、临盘状态标签 |
| 主文字 | `#F4F8F7` | 标题与正文 |
| 次文字 | `#AAB8B5` | 辅助信息、时间与元数据 |
| 风险提示 | `#FFBF7C` | 未核验、需注意；不能用作交易引导 |
| 错误/失败 | `#FF8D9C` | 转写失败、链接不可用 |
| 字体 | `Google Sans`, `PingFang SC`, `Microsoft YaHei`, sans-serif | 中文使用系统回退，数字和英文优先 Google Sans |
| 圆角 | 12 / 16 / 20px | 控件 / 卡片 / 主面板 |
| 阴影 | `0 18px 60px rgba(0,0,0,.28)` | 面板悬浮，不使用硬黑描边 |

### 2.3 字体与密度

- 页面标题：28px / 36px，Medium；页面只保留一个主标题。
- 模块标题：16px / 24px，Medium。
- 正文与字幕：14px / 22px；字幕阅读区可放大到 16px / 28px。
- 元数据：12px / 18px；采用等宽数字以提高时间辨识度。
- 桌面端内容区最大行宽约 720px，长字幕避免满屏横向延展。

### 2.4 玻璃组件规则

| 组件 | 视觉与交互 |
| --- | --- |
| 主面板 | 半透明填充、20px 模糊、1px 低对比描边、20px 圆角；不在面板内再叠过多玻璃层 |
| 内容卡片 | 次级玻璃表面；悬浮时描边略亮、上移 2px；点击后进入明确的选中态 |
| 临盘卡片 | 左侧 3px 薄荷绿光带 + “临盘新增”标签；不整卡荧光填充 |
| 主按钮 | 薄荷绿实色、深色文字；只用于“打开原视频”“查看临盘新增”等主任务 |
| 次按钮 | 透明玻璃按钮 + 描边；用于筛选、稍后、标记已处理 |
| 标签 | 小尺寸胶囊；状态以文字和图标共同表达，不能只靠颜色 |
| 焦点状态 | 2px `#8AFFC4` 外环，键盘导航必须可见 |

## 3. 信息架构与全局框架

### 3.1 一级导航

1. 今日简报
2. 博主时间线
3. 主题洞察
4. 观察清单
5. 决策记录
6. 设置

### 3.2 桌面布局（≥1280px）

采用稳定的三栏工作台，避免用户在临盘时频繁跳转。

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│ Logo  今日 · 07/17（周四）              全局搜索             账户 / 设置  │
├───────────────┬────────────────────────────────────────┬────────────────────┤
│ 248px 左导航  │ 主工作区（min 620px，可滚动）            │ 320px 右侧情境栏  │
│               │                                        │                    │
│ · 今日简报    │ 页面标题 + 筛选 + 更新时间              │ 参考截止 15:00    │
│ · 博主时间线  │                                        │  倒计时 00:22:18  │
│ · 主题洞察    │ 当前页面内容                            │ ───────────────── │
│ · 观察清单    │                                        │ 临盘新增 / 待处理 │
│ · 决策记录    │                                        │ 来源状态 / 风险   │
│               │                                        │                    │
│ 观察清单摘要  │                                        │                    │
└───────────────┴────────────────────────────────────────┴────────────────────┘
```

- 顶部栏高度 64px，采用透明玻璃并在滚动时增强背景模糊。
- 左侧栏固定；主工作区是唯一默认纵向滚动区域；右侧情境栏在桌面端固定。
- 右栏始终显示倒计时、临盘新增数和未处理数，避免用户离开今日简报后失去关键上下文。
- 页面最大可用宽度 1600px；大屏两侧保留至少 24px 呼吸空间。

### 3.3 自适应规则

| 宽度 | 布局 |
| --- | --- |
| ≥1280px | 三栏完整工作台 |
| 1024-1279px | 左侧栏收窄至 72px 图标栏；右栏变为可收起的 320px 抽屉 |
| 768-1023px | 两栏：顶部横向导航 + 主区；临盘状态固定为右上角状态胶囊，详情通过抽屉打开 |
| <768px | 非首发重点；改为单列阅读，底部导航，原视频与字幕上下排列 |

## 4. 核心页面布局

### 4.1 今日简报（默认首页）

用途：在 3 分钟内完成当天收盘前信息扫读。

```text
主工作区
┌───────────────────────────────────────────────────────────────┐
│ 今日收盘前简报                         上次同步 14:58 · 筛选  │
│ 3 条新增 · 2 条待处理 · 1 条需要核验                           │
├───────────────────────────────────────────────────────────────┤
│ [临盘新增] 14:56  博主 A  · 半导体            距截止 00:04:12 │
│ 一句话摘要（中性）                                           → │
│ 观点标签  已核验/待核验  字幕状态  [原视频] [查看详情]          │
├───────────────────────────────────────────────────────────────┤
│ [新增] 14:42  博主 B · 医药                                    │
│ …                                                               │
├───────────────────────────────────────────────────────────────┤
│ 今日主题概览： 半导体 2（观点分歧）  医药 1  黄金 1             │
└───────────────────────────────────────────────────────────────┘
```

- 首屏只显示临盘新增与最高相关的 3-5 条，旧内容进入“待处理”折叠区。
- 临盘卡片固定在普通新内容之前；用户已读后可折叠，但右栏仍保留计数。
- 任何“观点”均使用中性卡片，旁边紧邻来源、发布时间和风险标识；不设计“买入/卖出”按钮。

### 4.2 博主时间线

用途：按单一博主回看所有已入库视频，并识别其临盘发言。

```text
┌───────────────────────────────────────────────────────────────┐
│ ← 博主 A                    [近 7 天 ▾] [字幕搜索____________] │
│ 头像 · 平台认证状态 · 关注主题：半导体 / AI · 最近同步 14:58    │
│ [全部 24] [临盘 3] [未处理 4] [有字幕 22]                       │
├───────────────────────────────────────────────────────────────┤
│ 07/17  周四                                                     │
│ ──● [临盘新增] 14:56 发布 · 14:57 入库 · 03:18 · 转写完成      │
│    视频标题                                                      │
│    “02:14 …半导体…”                                             │
│    [播放原视频] [打开字幕] [标记已处理]                         │
│ ──○ 14:42 发布 · 14:43 入库 · 05:32 · 转写中 68%               │
│    视频标题                                                      │
│    [查看转写进度] [打开原视频]                                  │
├───────────────────────────────────────────────────────────────┤
│ 07/16  周三                                                     │
│ ──○ 11:20 发布 · 11:23 入库 · 04:08 · 已处理                   │
└───────────────────────────────────────────────────────────────┘
```

- 时间线使用细纵线和时间节点，而不是瀑布流，突出“何时发布/何时发现”。
- 首次显示近 7 天，日期范围可扩展；“全部”仅代表系统已合法入库的内容，不能暗示平台全部历史内容已同步。
- 视频卡片必须同时显示 `发布时间` 和 `入库时间`。两者差值是发现延迟的透明度信息。
- 字幕搜索定位命中内容，并将命中词以薄荷绿背景高亮；不改变原字幕含义。

### 4.3 视频详情：原视频与字幕并读

用途：核验摘要、快速回看关键片段、处理转写错误。

```text
┌───────────────────────────────────────┬───────────────────────────────┐
│ 原视频与关键信息                       │ 字幕 / 文本检索                │
│                                       │ [搜索“半导体”__________]       │
│ ┌───────────────────────────────────┐ │ ┌───────────────────────────┐ │
│ │ 平台原视频播放器或外链预览区       │ │ │ 00:00–00:12  …             │ │
│ │ （不可嵌入时展示原平台打开按钮）   │ │ │ 02:14–02:27  …半导体…      │ │
│ └───────────────────────────────────┘ │ │ 02:28–02:46  …             │ │
│ 14:56 发布 · 14:57 入库 · 03:18      │ └───────────────────────────┘ │
│ [在原平台打开] [复制链接] [报告问题] │ 转写完成 14:58 · 置信度 98%    │
├───────────────────────────────────────┴───────────────────────────────┤
│ 结构化阅读：创作者观点 | 可核验事实 | 风险与不确定性 | 关联主题        │
└───────────────────────────────────────────────────────────────────────┘
```

- 桌面端使用 `60:40` 左右分栏；视频与字幕独立滚动，用户可在视频播放时阅读字幕。
- 点击字幕段落：若平台播放器支持，跳转到对应时间；不支持时复制时间点并提示用户在原平台查看。
- 原视频不可嵌入时，左区不留空白：展示封面、标题、平台图标、发布时间和明显的“在原平台打开”按钮。
- 底部“结构化阅读”必须在原视频/字幕之后，避免用户把 AI 摘要误认为原始事实。

### 4.4 转写任务状态与异常页

参考 `/Users/xcyy/work/douyin_dataset/audioconvert_automation.py` 的可用流程，界面将转写处理视为异步任务：链接/音频信息校验 → 创建任务 → 轮询进度 → 取得文本 → 生成结构化摘要。任务状态如下：

| 状态 | 卡片显示 | 用户动作 |
| --- | --- | --- |
| 等待处理 | 灰色状态点 + “已入队” | 打开原视频；无需等待页面刷新 |
| 转写中 | 薄荷绿进度条 + 百分比 + 开始时间 | 继续浏览；可手动刷新状态 |
| 转写完成 | “字幕已生成” + 完成时间 + 置信度 | 阅读、搜索、纠错 |
| 低置信度 | 橙色“建议核对” + 受影响片段 | 打开原视频核验；报告错字 |
| 转写失败 | 红色“字幕暂不可用”，展示失败时间和可重试条件 | 打开原视频；在权限允许时重试 |
| 不可处理 | 中性“仅原视频可用” + 原因（无授权/无音轨/链接失效） | 打开原视频或移除内容 |

**临盘规则：** 视频刚入库时，内容卡片先展示标题、发布时间、来源和“字幕处理中”；不得因为字幕尚未完成而延迟临盘提醒。完成字幕后，卡片以非打断式方式更新。

### 4.5 观察清单

- 左侧是博主列表：头像、昵称、平台、重点开关、最近发布/同步时间、未处理数。
- 右侧是所选博主的设置：关联主题、通知级别、是否启用 14:50-15:00 临盘提醒、数据连接状态。
- “重点博主”开关要有确认文案：该博主临盘发布新内容时将使用独立提醒额度；这不代表其观点可信度更高。

## 5. 关键微交互

| 场景 | 设计行为 |
| --- | --- |
| 14:50 后有新视频 | 右上角临盘计数从 0 变为 1；卡片淡入并置顶；浏览器通知只显示来源、发布时间与剩余时间 |
| 转写进度更新 | 时间线卡片内进度条平滑更新；不弹出遮挡工作区的模态框 |
| 点击字幕 | 当前段落获得薄荷绿左边线和弱背景；播放器支持时同步跳转 |
| 搜索字幕 | 返回上下文片段、时间戳和视频卡；关键词高亮但保留完整句子 |
| 视频链接失效 | 保留原始发布时间、入库时间和此前摘要，标出“原链接不可用”及最后验证时间 |
| 收盘到 15:00 | 倒计时变为“交易窗口已结束”；停止临盘强调样式，时间线与复盘仍可正常访问 |

## 6. 可用性与无障碍要求

1. 正文与玻璃背景至少满足 4.5:1 对比度；字幕阅读区使用比普通卡片更不透明的背景。
2. 所有颜色状态均配合图标和文字，如“临盘新增”“待核验”“转写失败”。
3. `backdrop-filter` 不支持时，退化为不透明深色面板；功能与可读性不受影响。
4. 支持键盘导航、可见焦点环、字幕字号调节和减少动态效果偏好。
5. 倒计时与临盘提醒不是唯一信息通道，不能自动播放声音或强制抢占焦点。

## 7. 首版页面优先级

| 优先级 | 页面/能力 | 上线目的 |
| --- | --- | --- |
| P0 | 今日简报、博主时间线、视频详情（原链接 + 转写状态 + 字幕）、观察清单 | 覆盖“临盘不漏看、日常可回溯”的核心闭环 |
| P0 | 浏览器通知与右栏临盘状态 | 让用户不必反复刷新页面 |
| P1 | 主题洞察、跨博主字幕搜索、字幕纠错后台 | 提升横向比较与信息质量 |
| P2 | 复杂可视化、历史复盘图表、个性化布局 | 在核心使用稳定后再投入 |

## 8. 交付给开发前需确认

1. 可合法播放/嵌入各内容平台原视频的方式与限制。
2. 可合法送入转写服务的音频来源、用户授权与字幕保留期限。
3. 浏览器通知权限申请时机：建议在用户完成观察清单与时间窗设置后请求，不在首次打开页面时弹窗。
4. 14:50-15:00 实际监测频率、转写服务时延及失败重试策略；“新视频发现”和“字幕可用”应是两套独立 SLA。
